### 1. Nine Pillars Understanding

**Q1: Why is using AI Development Agents (like Gemini CLI) for repetitive setup tasks better for me growth as a system architect?**

**Q2: How do the Nine Pillars of AIDD help a developer grow into an M-Shaped Developer?**

### 2. Vibe Coding vs Specification-Driven Development

**Q1: Why does Vibe Coding usually create problems after one week?**  
A : Vibe Coding is fast and unstructured. Without clear specifications, code often becomes **messy, hard to maintain, and full of hidden bugs** as the project grows.

**Q2: How would Specification-Driven Development prevent those problems?**  
A : Specification-Driven Development begins with **clear requirements and system design**. This ensures the project stays **organized, maintainable, scalable**, and reduces errors over time.

---

### 3. Architecture Thinking

**Q1: How does architecture-first thinking change the role of a developer in AIDD?**  
A : Instead of just writing code line by line, developers **plan systems, guide AI tools, and make strategic decisions**. Coding becomes a smaller part of the role, while **designing and orchestrating the system takes priority**.

**Q2: Why must developers think in layers and systems instead of raw code?**  
A : Thinking in layers ensures the system is organized, scalable, and easy to maintain. Just coding line by line can create messy software.
